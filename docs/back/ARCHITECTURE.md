## 1. Глобальная архитектура (Backend + Frontend)

### Бэкенд (NestJS + Prisma)

- **База данных:** PostgreSQL (локальная, отказались от Docker для упрощения).
- **ORM:** Prisma (генерация клиента, миграции).
- **Структура:** Монолитное API с четким разделением на модули (Users, Posts, Events, Shop, Auth).
- **Авторизация:** JWT (Passport.js) + RolesGuard (разграничение прав по ролям).
- **Хранение файлов:** Локальный диск (папка `uploads/`), раздача статики через NestJS. В будущем — перенос в S3-хранилище.

### Фронтенд (React + Vite)

- **Стейт-менеджер:** Zustand (для пользователя и корзины).
- **Запросы:** React Query + Axios (кеширование, перехватчики для JWT).
- **Роутинг:** React Router v6.
- **Формы:** Обычный `useState` (пока без React Hook Form, чтобы не усложнять).
- **Архитектура:** FSD (Feature-Sliced Design) — внедряется постепенно, чтобы не перегружать проект на старте.

## 2. Функциональные требования (Фичи)

### 2.1. Роли пользователей (три сущности)

Мы отказались от общей таблицы `User` с одной ролью и разделили бизнес-сущности, но оставили общую таблицу `User` для авторизации.

| Сущность                          | Роль              | Назначение                                                                                          |
| :-------------------------------- | :---------------- | :-------------------------------------------------------------------------------------------------- |
| **Клубный участник** (ClubMember) | `CLUB_MEMBER`     | Посетитель клуба. Покупает абонементы, ходит на тренировки, оставляет отзывы.                       |
| **Тренер** (Trainer)              | `TRAINER`         | Проводит тренировки (индивидуальные/групповые), ведет клиентов, создает новости о занятиях.         |
| **Сотрудник** (Employee)          | `HR` / `DIRECTOR` | Администрация. Управляет справочниками (должности/специальности), создает опросы, смотрит дашборды. |

_Решение:_ У каждой сущности свой набор полей (специальность у тренера, должность у сотрудника), но вход в систему единый (через `User.email` и `User.password`).

### 2.2. Модули и их наполнение

#### А. Модуль "Новости и Лента" (News)

**Кто создает:** Тренеры и Администрация (HR/Director).  
**Кто читает:** Все (участники, тренеры, админы).  
**Фичи:**

- Публикация новостей с текстом и картинкой.
- Возможность привязать к новости **событие (тренировку)**.
- Комментарии под новостями (доступны всем авторизованным).
- Кнопка "Записаться" в один клик (если новость анонсирует тренировку).

#### Б. Модуль "Календарь и Тренировки" (Events)

**Кто создает:** Тренеры.  
**Кто участвует:** Клубные участники.  
**Фичи:**

- Создание тренировок (индивидуальных и групповых) с указанием времени, зала, максимального числа участников.
- Отображение количества свободных мест в реальном времени.
- Участник видит свои предстоящие тренировки в личном календаре.

#### В. Модуль "Отзывы о тренерах" (Trainer Reviews)

**Кто создает:** Клубные участники.  
**Кто смотрит:** Все пользователи (отображаются на странице тренера).  
**Фичи:**

- Оценка тренера (рейтинг).
- Текстовый отзыв.

#### Г. Модуль "Интернет-магазин мерча" (Shop)

**Кто создает товары:** Администрация.  
**Кто покупает:** Клубные участники.  
**Фичи:**

- Каталог товаров (фото, цена, наличие на складе).
- Корзина (Zustand).
- Оформление заказа (без реальной оплаты, статусы: "Новый" → "Оплачен" → "Отправлен").
- Админка: управление товарами (CRUD) и просмотр списка заказов.

#### Д. Модуль "Опросы" (Polls)

**Кто создает:** Администрация (HR/Director).  
**Кто отвечает:** Клубные участники.  
**Фичи:**

- Создание опроса с вариантами ответов.
- Голосование.
- Отображение статистики голосования.

#### Е. Модуль "Дашборды и Аналитика"

**Кто смотрит:** Только администрация (HR/Director).  
**Метрики:**

- Количество новых участников по месяцам.
- Топ тренеров по загрузке (количество клиентов).
- Золотое время (в какие дни/часы самые популярные тренировки).
