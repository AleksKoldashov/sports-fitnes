# Модуль "Клубный участник" (Club Member)

## 1. Общая концепция

**Клубный участник** — это основная сущность системы.
После входа участник попадает в **единый личный кабинет** (`/club-member`), где объединены управление профилем, тренировками и лента новостей.

## 3. Модель данных (Prisma)

### 3.1. Модель `ClubMember`

```prisma
model ClubMember {
  id          Int      @id @default(autoincrement())

  // Личные данные (заполняет участник)
  firstName   String
  lastName    String
  patronymic  String?
  age         Int
  phone       String?
  vk          String?
  telegram    String?

  // Данные, назначаемые тренером
  fitnessLevel FitnessLevel? @default(BEGINNER)
  nutritionPlan String?      // текстовое описание или ссылка на PDF

  // Статусы
  membershipStatus MembershipStatus? @default(TRIAL)
  membershipExpiresAt DateTime?

  // Связи
  userId      Int      @unique
  user        User     @relation(fields: [userId], references: [id])

  // Тренер, закреплённый за участником
  trainerId   Int?
  trainer     Trainer? @relation(fields: [trainerId], references: [id])

  // Интересы (многие-ко-многим)
  interests ClubMemberInterest[]

  // Записи на тренировки
  events     EventParticipant[]

  // Запросы на тренировки
  trainingRequests TrainingRequest[]

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

// ============================================
// ВСПОМОГАТЕЛЬНЫЕ ENUM'ы
// ============================================

enum FitnessLevel {
  BEGINNER
  INTERMEDIATE
  ADVANCED
  PRO
}

enum MembershipStatus {
  TRIAL
  ACTIVE
  EXPIRED
}
````

## 4. Процессы и сценарии

### 4.1. Регистрация участника

1. Пользователь заполняет форму: `email`, `password`, `firstName`, `lastName`, `age`.
2. Бэкенд создаёт `User` (роль `CLUB_MEMBER`) и `ClubMember`.
3. Участник получает доступ к системе (JWT).
4. **После входа участник попадает на страницу `/club-member` (личный кабинет).**

---

### 4.2. Заполнение профиля

1. Участник находится на странице `/club-member` (левая колонка).
2. Может заполнить:
   - `phone`, `vk`, `telegram`.
   - Выбрать интересы (из списка).
3. Все изменения отправляются через `PATCH /club-member/profile`.

---

### 4.3. Закрепление тренера за участником

1. **Тренер** в админке видит список свободных участников.
2. Нажимает "Закрепить за собой" → `ClubMember.trainerId` = ID тренера.
3. Участник видит имя своего тренера в левой колонке `/club-member`.

---

### 4.4. Назначение уровня подготовки и плана питания

1. **Тренер** открывает карточку участника.
2. Заполняет поля:
   - `fitnessLevel` (из выпадающего списка).
   - `nutritionPlan` (текст или ссылка).
3. Данные сохраняются через `PATCH /club-member/profile` (доступ только у тренера).

---

### 4.5. Запрос участника на тренировку

1. Участник в левой колонке нажимает "Запрос на тренировку".
2. Заполняет форму:
   - Тип: `INDIVIDUAL` или `GROUP`.
   - Предпочтительное время.
   - Комментарий (опционально).
3. Отправляет запрос через `POST /training-requests`.
4. Запрос попадает тренеру в список `PENDING`.
5. Тренер принимает или отклоняет запрос.

---

### 4.6. Подтверждение тренировки участником

1. Тренер создаёт тренировку (`Event`) со статусом `PENDING_CONFIRMATION`.
2. Участник видит её в левой колонке в разделе "Мои тренировки".
3. Участник нажимает "Подтвердить" или "Отказаться".
4. Статус участия в `EventParticipant` меняется на `CONFIRMED` или `DECLINED`.

---

### 4.7. Отметка посещаемости

1. После тренировки **тренер** отмечает каждого участника:
   - `PRESENT` (был).
   - `ABSENT` (не пришёл).
   - `LATE` (опоздал).
2. Статус события меняется на `COMPLETED`.
3. Участник видит историю посещений в левой колонке.

---

### 4.8. Взаимодействие с лентой новостей (правая колонка)

1. Участник видит в правой колонке:
   - **Новости клуба** (от администрации).
   - **Анонсы тренировок** (от тренеров).
   - **Посты тренеров** (советы, рекомендации).
2. Если в анонсе есть кнопка "Записаться" — она ведёт на создание запроса на тренировку или на подтверждение уже созданного события.

## 5. API эндпоинты

### 5.1. Профиль участника

| Метод   | Эндпоинт               | Доступ                   | Описание                                        |
| :------ | :--------------------- | :----------------------- | :---------------------------------------------- |
| `GET`   | `/club-member/profile` | `CLUB_MEMBER`            | Получить свой профиль                           |
| `PATCH` | `/club-member/profile` | `CLUB_MEMBER`, `TRAINER` | Обновить профиль (разные поля для разных ролей) |

---

### 5.2. Интересы

| Метод    | Эндпоинт                     | Доступ        | Описание                       |
| :------- | :--------------------------- | :------------ | :----------------------------- |
| `GET`    | `/interests`                 | Все           | Получить список всех интересов |
| `POST`   | `/club-member/interests`     | `CLUB_MEMBER` | Добавить интересы (массив ID)  |
| `DELETE` | `/club-member/interests/:id` | `CLUB_MEMBER` | Удалить интерес                |

---

### 5.3. Запросы на тренировки

| Метод   | Эндпоинт                        | Доступ        | Описание                               |
| :------ | :------------------------------ | :------------ | :------------------------------------- |
| `POST`  | `/training-requests`            | `CLUB_MEMBER` | Создать запрос на тренировку           |
| `GET`   | `/training-requests`            | `TRAINER`     | Получить список запросов (для тренера) |
| `PATCH` | `/training-requests/:id/accept` | `TRAINER`     | Принять запрос                         |
| `PATCH` | `/training-requests/:id/reject` | `TRAINER`     | Отклонить запрос                       |

---

### 5.4. Тренировки

| Метод   | Эндпоинт                   | Доступ        | Описание                         |
| :------ | :------------------------- | :------------ | :------------------------------- |
| `GET`   | `/events`                  | Все           | Список тренировок (с фильтрами)  |
| `POST`  | `/events`                  | `TRAINER`     | Создать тренировку               |
| `PATCH` | `/events/:id/confirm-user` | `CLUB_MEMBER` | Подтвердить участие              |
| `PATCH` | `/events/:id/decline-user` | `CLUB_MEMBER` | Отказаться от участия            |
| `PATCH` | `/events/:id/attendance`   | `TRAINER`     | Отметить посещаемость участников |
| `PATCH` | `/events/:id/complete`     | `TRAINER`     | Завершить тренировку             |
