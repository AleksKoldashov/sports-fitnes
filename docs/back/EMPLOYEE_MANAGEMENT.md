````markdown
# Управление сотрудниками (HR & Admin)

## 1. Общая архитектура

В системе реализовано **два способа** создания сотрудников:

| Способ              | Кто создаёт   | Процесс                                                    | Статус                         |
| :------------------ | :------------ | :--------------------------------------------------------- | :----------------------------- |
| **Прямое создание** | Директор      | Сразу создаёт пользователя с нужной ролью                  | Сотрудник активен сразу        |
| **Через заявку**    | Кадровик (HR) | Создаёт заявку → Директор утверждает → Создаётся сотрудник | Заявка проходит жизненный цикл |

---

## 2. Доступы и права

### Матрица доступов

| Действие                            |    HR (Кадровик)    |    Директор     |
| :---------------------------------- | :-----------------: | :-------------: |
| Создать сотрудника напрямую         |  ❌ (Нет доступа)   | ✅ (Сразу в БД) |
| Создать заявку на сотрудника        | ✅ (Статус PENDING) |       ❌        |
| Просматривать все заявки            |  ❌ (только свои)   |    ✅ (все)     |
| Просматривать свои заявки           |         ✅          |       ❌        |
| Утвердить заявку                    |         ❌          |       ✅        |
| Отклонить заявку (с комментарием)   |         ❌          |       ✅        |
| Отозвать заявку (в статусе PENDING) |         ✅          |       ❌        |

---

## 3. Жизненный цикл заявки (Статусы)

Заявка проходит следующие статусы:

```text
[Создание HR]
    ↓
PENDING (Ожидает)
    ↓
    ├── APPROVED (Утверждена) → Создан пользователь
    │
    ├── REJECTED (Отклонена) → Указана причина отказа
    │
    └── CANCELLED (Отозвана) → HR отозвал до рассмотрения
```
````

### Описание статусов

| Статус      | Описание                                  | Кто может изменить     |
| :---------- | :---------------------------------------- | :--------------------- |
| `PENDING`   | Заявка создана, ожидает решения директора | —                      |
| `APPROVED`  | Заявка утверждена, пользователь создан    | Директор               |
| `REJECTED`  | Заявка отклонена с комментарием           | Директор               |
| `CANCELLED` | Заявка отозвана кадровиком                | HR (только из PENDING) |

---

## 4. Эндпоинты API

### 4.1. Прямое создание сотрудника (Директор)

**Endpoint:** `POST /admin/employees/direct`

**Доступ:** только `DIRECTOR`

**Тело запроса:**

```json
{
  "email": "hr@club.ru",
  "password": "hr123",
  "role": "HR",
  "firstName": "Мария",
  "lastName": "Смирнова",
  "patronymic": "Алексеевна",
  "phone": "+7 999 123-45-67"
}
```

**Поля, обязательные для разных ролей:**

| Роль       | Обязательные поля                                                           | Опциональные поля          |
| :--------- | :-------------------------------------------------------------------------- | :------------------------- |
| `TRAINER`  | `firstName`, `lastName`, `email`, `password`, `role`, `specialty`           | `patronymic`, `experience` |
| `MANAGER`  | `firstName`, `lastName`, `email`, `password`, `role`, `department`, `phone` | `patronymic`               |
| `HR`       | `firstName`, `lastName`, `email`, `password`, `role`, `phone`               | `patronymic`               |
| `DIRECTOR` | `firstName`, `lastName`, `email`, `password`, `role`                        | `patronymic`               |

**Ответ (201 Created):**

```json
{
  "id": 2,
  "email": "hr@club.ru",
  "role": "HR"
}
```

---

### 4.2. Создание заявки (HR)

**Endpoint:** `POST /admin/applications`

**Доступ:** только `HR`

**Тело запроса:**

```json
{
  "email": "trainer@club.ru",
  "role": "TRAINER",
  "firstName": "Пётр",
  "lastName": "Кузнецов",
  "patronymic": "Иванович",
  "specialty": "Йога, Пилатес",
  "experience": 5
}
```

**Ответ (201 Created):**

```json
{
  "id": 1,
  "email": "trainer@club.ru",
  "role": "TRAINER",
  "status": "PENDING",
  "createdById": 2
}
```

---

### 4.3. Получение списка заявок

**Endpoint:** `GET /admin/applications`

**Доступ:** `HR` (только свои) и `DIRECTOR` (все)

**Query-параметры (все опциональные):**

| Параметр | Тип    | Описание                      | Пример                                         |
| :------- | :----- | :---------------------------- | :--------------------------------------------- |
| `status` | string | Фильтр по статусу заявки      | `pending`, `approved`, `rejected`, `cancelled` |
| `limit`  | number | Количество заявок на странице | `10` (по умолчанию)                            |
| `offset` | number | Сдвиг для пагинации           | `0` (по умолчанию)                             |

**Пример запроса:**

```bash
GET /admin/applications?status=pending&limit=5&offset=0
```

**Ответ (200 OK):**

```json
{
  "data": [
    {
      "id": 1,
      "email": "trainer@club.ru",
      "role": "TRAINER",
      "status": "PENDING",
      "profileData": {
        "firstName": "Пётр",
        "lastName": "Кузнецов",
        "patronymic": "Иванович",
        "specialty": "Йога, Пилатес",
        "experience": 5
      },
      "createdAt": "2026-07-08T10:00:00.000Z",
      "createdBy": {
        "id": 2,
        "email": "hr@club.ru"
      }
    }
  ],
  "pagination": {
    "total": 1,
    "limit": 5,
    "offset": 0,
    "hasMore": false
  }
}
```

---

### 4.4. Утверждение / Отклонение заявки (Директор)

**Endpoint:** `POST /admin/applications/approve`

**Доступ:** только `DIRECTOR`

**Тело запроса:**

```json
{
  "applicationId": 1,
  "action": "APPROVE" // или "REJECT"
}
```

**Для отклонения обязательно указать причину:**

```json
{
  "applicationId": 1,
  "action": "REJECT",
  "rejectionReason": "Недостаточный опыт работы (требуется минимум 3 года)"
}
```

**Ответ (200 OK) при APPROVE:**

```json
{
  "message": "Заявка утверждена, сотрудник создан"
}
```

**Ответ (200 OK) при REJECT:**

```json
{
  "id": 1,
  "status": "REJECTED",
  "rejectionReason": "Недостаточный опыт работы (требуется минимум 3 года)",
  "reviewedById": 1
}
```

---

### 4.5. Отзыв заявки (HR)

**Endpoint:** `PATCH /admin/applications/:id/cancel`

**Доступ:** только `HR` (только свои заявки в статусе `PENDING`)

**Пример запроса:**

```bash
PATCH /admin/applications/1/cancel
```

**Ответ (200 OK):**

```json
{
  "id": 1,
  "status": "CANCELLED",
  "updatedAt": "2026-07-08T12:00:00.000Z"
}
```

## 5. Ошибки и их обработка

| Код | Описание                                               | Когда возникает                                         |
| :-- | :----------------------------------------------------- | :------------------------------------------------------ |
| 400 | `Пользователь с такой почтой уже существует`           | При попытке создать дубликат                            |
| 400 | `Некорректный статус`                                  | Передан неверный статус в query                         |
| 403 | `Только директор может создавать сотрудников напрямую` | Попытка HR создать напрямую                             |
| 403 | `Только HR может создавать заявки`                     | Попытка директора создать заявку                        |
| 403 | `Только директор может утверждать заявки`              | Попытка HR утвердить заявку                             |
| 404 | `Заявка не найдена`                                    | Передан неверный ID заявки                              |
| 409 | `Заявка уже обработана`                                | Попытка утвердить/отклонить заявку не в статусе PENDING |

## 6. Связь с другими модулями

- **AuthModule:** Используется для проверки JWT и получения текущего пользователя.
